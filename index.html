Why Should You Choose SHKeeper?

It's free and open-source

SHKeeper is a self-hosted keeper, an open-source cryptocurrency payment processor. In more detail, SHKeeper is free and open-source, non-custodial, self-hosted software. It allows you to receive payments P2P directly to your software or hardware wallet. Being free and open-source it gives an excellent opportunity for developers to build things and integrations on top of SHKeeper Thus you've got yourself your own payment processor. Equipped with SHKeeper, technically, it is like your own bank in your pocket.

Privacy of the customers

SHKeeper offers a direct way for store-owners and charities to receive Bitcoin, Ethereum, LiteCoine, and other cryptocurrency payments, which are bound to improve the privacy of the customers/donors significantly. There are no subscriptions, no transaction fees. No third-party involvement significantly increases the censorship resistance, privacy, and security for you and your customers. More to it, SHKeeper enables you to become a processor yourself so that you can offer different packages and help spread the adoption locally or globally.

FAQ

Бесплатный процессор криптовалютных платежей с открытым исходным кодом, который позволяет вам получать платежи напрямую, без комиссий, транзакционных издержек или посредников. Это система выставления счетов, не связанная с хранением, которая исключает участие третьей стороны. Платежи поступают прямо на ваш кошелек, что повышает конфиденциальность и безопасность. Ваши закрытые ключи никогда не требуются для получения платежей на ваш сервер. Повторное использование адреса невозможно, так как каждый счет использует новый адрес для получения платежей на ваш кошелек.

Сервер SHKeeper — это многофункциональное программное обеспечение с множеством вариантов использования, которое может решить проблемы для разных типов пользователей. Это может быть полезно торговцам, создателям контента, пользователям сети Lightning, биржам, хостинг-провайдерам и многим другим.

Это зависит от того, какой из различных вариантов развертывания соответствует вашим конкретным потребностям.

Вы можете запустить свой сервер SHKeeper и быть обработчиком платежей для своих друзей или местного рынка. Другим вариантом использования является прием пожертвований через приложение POS (Point of Sale) или кнопки оплаты, которые можно скопировать и вставить в виде фрагментов HTML на любой веб-сайт. Также можете использовать SHKeeper для вашего онлайн магазина.

SHKeeper Server помогает устранить проблему конфиденциальности повторного использования адреса. Он делает это, предоставляя новый адрес каждый раз, когда запрашивается счет для оплаты. Все это делается автоматически, и продавцу не нужно отслеживать, какие адреса принадлежат какому кошельку, магазину и т. д. Сервер SHKeeper организует всю платежную информацию в подробной системе выставления счетов для продавца. Принцип работы довольно прост. Продавцы подключают кошелек к каждому из своих магазинов, в которые они хотят получать платежи. Счета, выставленные для платежей в магазине, напрямую связаны с подключенным кошельком продавца. Адреса счетов получены из xpubkey (opens new window) кошелька, связанного с магазином. Программному обеспечению нужен только ваш расширенный открытый ключ кошелька для создания новых адресов для каждого платежа. Эти адреса контролируются сервером SHKeeper по мере их перемещения.

Если у вас возникли проблемы с оплатой счета SHKeeper Server, вероятно, это связано с одной из следующих причин: ..........

Чтобы запросить возврат у продавца, вы должны связаться с продавцом напрямую! Сервер SHKeeper не имеет отношения к продавцу, у которого вы приобрели продукт или услугу. SHKeeper Server — это стек программного обеспечения с открытым исходным кодом, а не компания. Сообщество и участники SHKeeper не имеют никакого контроля над тем, кто использует программное обеспечение или как они его используют. Если вы оплатили счет продавцу и у вас возникла проблема с заказом, вы должны связаться с продавцом напрямую, чтобы узнать, что произошло. Каждый продавец, использующий программное обеспечение, контролирует свой собственный магазин и подключенные кошельки, на которые поступают средства. SHKeeper не владеет и не имеет доступа к каким-либо средствам магазина, использующего программное обеспечение SHKeeper, только продавец.

The original BTCPay Server API is compatible for the most part with BitPay's API (opens new window)to allow merchants to smoothly transition to using BTCPay if they prefer a free, open source, payment processing alternative. In 2020, BTCPay Server started releasing a new Greenfield API. This new API will co-exist with the original API and allow full usage of all BTCPay Server features, without requiring the UI. You can view the current Greenfield API documentation (opens new window). A BTCPay Server functionality that is not found in the Greenfield API documentation means it isn't fully implemented yet in the new API and users should use the original API instead. The discussion on the development of the new Greenfield API can be found here (opens new window).
